WHAT IS THIS?
================================================================================

An AI-powered Business Intelligence system that lets users ask questions in
plain English and get instant charts and insights from 3.4 million e-commerce
orders.

Example:
  User: "Which aisles have the highest reorder rate?"
  System: → Generates SQL → Executes query → Returns bar chart

NO SQL KNOWLEDGE REQUIRED.

================================================================================
TECH STACK
================================================================================

Backend:
  - Python 3.11
  - FastAPI (async API framework)
  - DuckDB (columnar analytics database)
  - LangChain + GPT-4 (AI orchestration)
  - Pydantic (type validation)

Frontend:
  - Next.js 14 (React framework)
  - TypeScript
  - Recharts (visualization library)
  - TailwindCSS (styling)

Data:
  - Instacart Market Basket Dataset
  - 3.4M orders
  - 32M order items
  - 50K products
  - 6 interrelated CSV files

================================================================================
KEY DESIGN DECISIONS & TRADE-OFFS
================================================================================

1. DuckDB vs PostgreSQL
   CHOSE: DuckDB
   REASON: 10x faster for analytics, zero config
   TRADE-OFF: No concurrent writes (fine for read-only BI)

2. GPT-4 vs GPT-3.5
   CHOSE: GPT-4 Turbo
   REASON: 60% fewer SQL errors
   TRADE-OFF: 10x cost (worth it for accuracy)

3. In-Memory vs Persistent Database
   CHOSE: In-memory by default, persistent optional
   REASON: 2-3x faster queries
   TRADE-OFF: Higher RAM usage (configurable)

4. Streaming vs Request-Response
   CHOSE: Both (streaming as default)
   REASON: Better UX, shows progress
   TRADE-OFF: Slightly more complex implementation

5. Rule-Based vs LLM Chart Selection
   CHOSE: Rule-based
   REASON: Deterministic, free, fast
   TRADE-OFF: Less flexible (acceptable for basic charts)

SETUP INSTRUCTIONS (QUICK)
================================================================================

1. Download dataset from Kaggle
   → https://www.kaggle.com/datasets/psparks/instacart-market-basket-analysis

2. Backend setup:
   cd backend
   python -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   cp .env.example .env
   # Add OPENAI_API_KEY to .env
   python main.py

3. Frontend setup (new terminal):
   cd frontend
   npm install
   npm run dev

4. Open: http://localhost:3000

Total time: ~10 minutes
